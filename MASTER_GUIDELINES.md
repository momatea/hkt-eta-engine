# 🛰️ HKT-Radar-Engine: Master Guidelines & Rules

เอกสารนี้รวบรวม "กฎเหล็ก (Logic Locks)" ทั้งหมดที่ผ่านการทดสอบแก้บั๊กหน้างานจริงมาแล้ว ห้าม AI หรือผู้พัฒนาแก้โค้ดเปลี่ยนตรรกะเหล่านี้เด็ดขาด เพื่อป้องกันความแม่นยำ 100% พังทลาย

## 🛑 Strict Rules (กฎเหล็กลอจิกเรดาร์)

1. **AOBT Precision (The 15m Rule)**: 
   - เวลาถอย (Pushback) ที่แท้จริงคือวินาทีแรกที่เครื่องขยับล้อพ้น **15 เมตร** (First Step)
   - เครื่องต้องขยับออกไปถึง 55 เมตร และอยู่นิ่งครบ 5 รอบ ถึงจะถือว่า "คอนเฟิร์มการถอย"
   - เมื่อคอนเฟิร์มแล้ว ให้ **ย้อนเวลากลับไป (Back-date)** ใช้เวลาตอน 15 เมตร เป็น AOBT ส่งให้ API ห้ามใช้เวลาตอนที่คอนเฟิร์มเด็ดขาด

2. **The Geofence Lock (สำคัญมาก ป้องกัน JQ28 Bug)**:
   - ตรรกะตรวจจับต้องเชื่อพิกัด (Geography) มากกว่า Metadata ของเรดาร์!
   - ถ้าตรวจพบไฟลท์ประเภทเอกชน/เช่าเหมาลำ (Whitelisted Carriers) ที่จอดนิ่งอยู่ หรือเริ่มขยับตัวอยู่ **"ภายในรัศมีหลุมจอด" (Inside Gate < Stand Radius)** 
   - ให้บังคับสถานะไฟลท์นั้นเป็น **ขาออก (DEPARTURE) เสมอ** แม้ว่าข้อมูลปลายทางจะยังค้างว่ากำลังเดินทางมาภูเก็ตก็ตาม เพื่อป้องกันระบบยิงเวลา `TOUCHDOWN` ผิดจังหวะตอนมันดันถอย

3. **Turnaround Handover (ระบบส่งกะ MH790 -> MH791)**:
   - เครื่องที่มีเลขรหัส ID ของ Flightradar เดิมบินมาลงแล้ว จะเปลี่ยนเป็นไฟลท์ขาออกได้ก็ต่อเมื่อ "จอดนิ่งสนิท (< 5 kts)", "เข้าจอดคาหลุมแล้ว (< 60m)", และ "ข้อมูลปลายทางเปลี่ยนไป" ถึงจะอนุญาตให้ดักจับ AOBT ได้ (ป้องกันไฟลท์ตีกัน)

4. **Stand Configuration**: 
   - รัศมีหลุมจอดในลาน A และ D ทั้งหมด ต้องฟิกซ์ไว้ที่ **80 เมตร** เท่านั้น
   - การดึงข้อมูล FR24 ต้องใช้ JSON API ซ่อนหลังบ้านเท่านั้น ห้ามใช้ Headless Browser เพื่อประหยัด RAM บน Server Render

## ✅ Known Issues & Safe to Update
- หากเรดาร์เกิดอาการ Jitter มากๆ ในบางเบย์ สามารถปรับแต่งตัวแปร `STABLE_REQUIRED` ให้เข้มข้นขึ้นได้
- หากต้องการเพิ่มสายการบินเช่าเหมาลำ สามารถใส่เพิ่มใน `CARRIER_WHITELIST` ได้อย่างปลอดภัย
