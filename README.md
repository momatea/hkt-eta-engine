# 🛫 HKT ETA Engine (For Line OA)

โปรเจกต์นี้ถูก Fork แยกออกมาจาก `hkt-radar-engine` ตัวเต็ม เพื่อนำมาทำเป็น **"ระบบดึงเวลา ETA ขาเข้า"** โดยเฉพาะ สำหรับส่งข้อมูลให้ Line OA 

เป้าหมายคือทำ API ที่ทำงานเร็วที่สุด เบาที่สุด และตอบกลับเฉพาะข้อมูลที่จำเป็น (Callsign และ ETA) โดยไม่ต้องสนใจเรื่องการจอดที่หลุม หรือการถอย (AOBT/AIBT) ใดๆ ทั้งสิ้น

---

## 🧠 ทบทวนความจำ: สถาปัตยกรรม "5 Zones" (The 5-Zone Radar Scan)

ในอดีต ตอนที่เราทำระบบดึงข้อมูล ETA ล่วงหน้า 3-4 ชั่วโมง เราเจอปัญหาว่า **Flightradar24 มีการจำกัดจำนวนไฟลท์สูงสุดต่อ 1 Bounding Box (กรอบสี่เหลี่ยมแผนที่)** 
ถ้าเราวาดกรอบสี่เหลี่ยมครอบจากภูเก็ตไปยันอินเดีย จีน และสิงคโปร์ในรอบเดียว FR24 จะส่งข้อมูลมาไม่ครบ (Truncate) ทำให้เรา **"ตกหล่น"** ไฟลท์ที่บินมา HKT ไปหลายลำ

**วิธีแก้ปัญหาของเราในตอนนั้น (ซึ่งต้องนำมาใช้ในโปรเจกต์นี้):**
1. **แบ่งเค้กเป็น 5 ส่วน (5 Zones):** เราหั่นแผนที่รอบภูเก็ตออกเป็น 5 Bounding Box ย่อยๆ (เช่น ทิศเหนือ, ทิศใต้, ทิศตะวันออก, ทิศตะวันตก, และกรอบใกล้สนามบิน)
2. **ตัวแปร `APPROACH_ZONES`:** ในโค้ดเราจะมี Array เก็บพิกัด `bounds=lat_max,lat_min,lon_min,lon_max` ทั้ง 5 กรอบนี้ไว้
3. **Sequential Polling:** เราให้เซิร์ฟเวอร์วนลูปยิงขอข้อมูลทีละโซน (หรือยิงพร้อมกันแบบขนาน `Promise.all`) เพื่อเก็บกวาดไฟลท์ที่ `destination === "HKT"` ให้ครบทุกทิศทาง โดยไม่ชนลิมิตของ FR24
4. **ETA Extraction:** เมื่อเจอไฟลท์ที่มาภูเก็ต เราจะเอา `flight.id` ไปยิงถาม API เจาะจง (`fetchFlight`) อีกรอบเพื่อดึงเวลา `detail.arrival` หรือ `detail.scheduledArrival` มาเป็น ETA

---

## 🛠️ แนวทางการพัฒนาโค้ด (สำหรับให้ Claude ทำต่อ)

1. **Clean Up:** ลบโค้ดตรรกะ Ground Scan, Geofence Lock, AOBT, AIBT, รัศมี 80 เมตร (hkt_stands.js) ทิ้งให้หมด เพราะไม่จำเป็นสำหรับ ETA
2. **Focus on Approach:** คงเหลือไว้แค่ระบบ `pollGroup(APPROACH_ZONES)` ที่วนลูป 5 โซนอย่างมีประสิทธิภาพ
3. **API Payload:** ปรับ Endpoint (เช่น `/api/flights/eta`) ให้ Return JSON ออกมาหน้าตาเรียบง่ายที่สุด เพื่อให้ Line OA ดึงไปใช้ต่อได้ทันที เช่น:
   ```json
   [
     { "Callsign": "TG201", "ETA": "2026-05-03T18:00:00+07:00" },
     { "Callsign": "FD3018", "ETA": "2026-05-03T18:15:00+07:00" }
   ]
   ```
4. **Rate Limiting:** อย่าลืมใส่ตัวหน่วงเวลา (Timeout/Delay) ในการดึงข้อมูล `fetchFlight` รายลำ เพื่อป้องกัน FR24 แบน IP ของเซิร์ฟเวอร์ (Render)
